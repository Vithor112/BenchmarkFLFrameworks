Compara escalabilidade (perfomance conforme o numero de clientes aumenta) e usabilidade dos frameworks de federated learning, OpenFL, NVFlare, FedML and Flower
OpenFL perfomou muito mal, e está descontinuado.
Os outros performaram bem. 

Ponto interessante: 
Furthermore, we have considered only the strong scalability, which provides
an analysis when the amount of operations needed decreases with an increasing
number of clients. On the other hand the weak scalability provides results when
the amount of work is constant and the number of clients increases. Treating
both strong and weak scalability would result in a more complete evaluation of
the FLFs

Além disso, escalabilidade está diretamente relacionado ao custo de comunicação entre cliente-servidor. 

Parametros utilizados: 
As backbone network we chose ResNet18 [13] trained from scratch using
the Adam optimizer with an initial learning rate set to 0.0001.A total of 100 FL
rounds were performed, with 1 epoch of local training performed by each client
using a batch size of 64.
